// 01 — About Me

코드로 문제를
해결하는 개발자

정보가 흩어져 있어 발생하는 소통의 지연과 비효율을 코드로 자동화하여 해결하는 것을 좋아합니다. 첫 프로젝트인 SIGNAL을 진행하며 여러 뉴스 매체에 흩어진 데이터를 자동으로 수집하는 환경을 만들고, 수집한 텍스트 데이터를 분석 모델과 연결하여 정량적인 위험 지표로 나타낼 수 있도록 구현했습니다.

단순히 데이터를 가져오는 것에 그치지 않고, 수집 중 발생하는 차단 문제나 인프라 보안 문제를 주도적으로 해결해 보았습니다. 전체적인 데이터 흐름을 직접 구현하고 튼튼하게 다지는 과정에서 큰 성장을 경험했으며, 앞으로도 기본기가 단단한 개발자로 성장하고자 합니다.

01

데이터 중심의 생각

개인의 주관적인 짐작 대신, 수집한 텍스트 데이터와 통계치를 바탕으로 객관적인 위험 기준을 정하고자 노력했습니다.

02

흐름에 대한 이해

웹 페이지 스크래핑부터 텍스트 데이터 분석, 데이터베이스 적재, 그리고 사용자가 보는 화면에 전달하기까지의 전체적인 흐름을 직접 연결해 보았습니다.

03

끝까지 해결하는 태도

단순 기능 구현에 그치지 않고 수집 서버 차단이나 데이터베이스 보안 노출 등, 프로젝트 중 마주친 크고 작은 문제들을 분석하며 우회책과 해결 방안을 찾았습니다.

Task 01 — 데이터 자동 수집
수집 안정성 확보 및 웹 차단 해결

[문제] 다량의 해외 매체 스크래핑 시 봇 탐지 시스템에 의해 임시 차단되거나 로딩 지연이 발생하는 문제 발생.
[해결] 웹 드라이버 우회 설정을 적용하고 주기적으로 수집을 반복하는 스케줄러를 등록하여 차단 없이 안정적으로 데이터를 가져오도록 개선했습니다.

Task 02 — 인프라 및 보안
안전한 데이터베이스 접근 환경 구축

[문제] 외부 데이터베이스를 연동하는 과정에서 포트 포워딩으로 공인 IP를 직접 노출할 시 발생하는 보안 위험성 인지.
[해결] 가상 사설망 서비스인 Tailscale을 도입하여, 외부 포트 노출 없이 기기 간 일대일 암호화 네트워크 환경을 구축하여 원격 적재를 안전하게 완료했습니다.

Task 03 — 데이터 정제 및 가공
분석 모델 활용 및 위험 지수 기준 수립

[문제] 일반적인 형태소 분석만으로는 경제·무역 특유의 부정적 단어(규제, 차질 등)를 완벽히 인지하기 어려운 한계 발견.
[해결] 한국어 자연어 처리 모델인 BERT를 활용해 기사 문맥을 파악하고, 자체 커스텀 단어 사전을 매핑한 뒤 수집 빈도를 점수화하여 위험 기준을 다듬었습니다.
